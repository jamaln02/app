# 🚀 دليل رفع موقع "روح" على Hostinger - خطوة بخطوة

## المرحلة الأولى: تحضير الملفات (5 دقائق)

### الخطوة 1: تنزيل ملفات المشروع من Emergent

**الطريقة الأولى: من واجهة Emergent**
1. في صفحة المشروع، ابحث عن زر **"Download"** أو **"Export"** (عادة في الأعلى)
2. اضغط عليه
3. سيتم تحميل ملف مضغوط (ZIP) على جهازك
4. فك ضغط الملف في مكان آمن على سطح المكتب

**الطريقة الثانية: إذا لم تجد زر Download**
- اطلب من الدعم تنزيل المشروع
- أو استخدم Git إذا متاح

### الخطوة 2: البحث عن مجلد build

بعد فك ضغط الملف:
1. افتح المجلد الذي تم تنزيله
2. ادخل على مجلد `frontend`
3. **هام جداً:** ابحث عن مجلد اسمه `build`
   - إذا **لم تجد** مجلد build، لا تقلق! سنعمله الآن

### الخطوة 3: إنشاء Build (إذا لم يكن موجوداً)

**⚠️ هذه الخطوة مهمة جداً!**

إذا لم تجد مجلد build، اتبع هذه الخطوات:

**على Windows:**
1. افتح Command Prompt (CMD)
   - اضغط Windows + R
   - اكتب `cmd` واضغط Enter
2. انتقل لمجلد المشروع:
   ```
   cd C:\path\to\your\project\frontend
   ```
3. نفذ هذه الأوامر:
   ```
   yarn install
   yarn build
   ```

**على Mac/Linux:**
1. افتح Terminal
2. انتقل لمجلد المشروع:
   ```
   cd /path/to/your/project/frontend
   ```
3. نفذ:
   ```
   yarn install
   yarn build
   ```

**النتيجة:** سيتم إنشاء مجلد `build` يحتوي على:
```
build/
├── index.html
├── static/
│   ├── css/
│   ├── js/
│   └── media/
└── asset-manifest.json
```

## المرحلة الثانية: الدخول إلى Hostinger (10 دقائق)

### الخطوة 4: تسجيل الدخول إلى Hostinger

1. **افتح المتصفح** (Chrome أو Firefox)
2. اذهب إلى: **https://hostinger.com**
3. اضغط على **"Login"** من الزاوية اليمنى العليا
4. أدخل:
   - **Email:** البريد الإلكتروني الذي سجلت به
   - **Password:** كلمة المرور
5. اضغط **"Sign In"**

### الخطوة 5: الوصول إلى لوحة التحكم (hPanel)

بعد تسجيل الدخول:
1. ستظهر لك صفحة بها جميع خدماتك
2. ابحث عن الاستضافة التي اشتريتها (Hosting)
3. اضغط على زر **"Manage"** أو **"إدارة"** بجانب اسم الاستضافة
4. ستنتقل إلى **hPanel** (لوحة التحكم)

**الشكل المتوقع:**
- ستجد قائمة جانبية على اليسار
- في الأعلى معلومات عن استضافتك
- في الوسط أدوات مختلفة

## المرحلة الثالثة: رفع الملفات (15 دقيقة)

### الخطوة 6: فتح File Manager (مدير الملفات)

1. في لوحة hPanel، ابحث عن قسم **"Files"** أو **"الملفات"**
2. اضغط على **"File Manager"** أو **"مدير الملفات"**
3. ستفتح نافذة جديدة أو تبويب جديد

**ستشاهد:**
- قائمة بالمجلدات على اليسار
- محتوى المجلد الحالي على اليمين
- أزرار في الأعلى (Upload, New Folder, Delete, إلخ)

### الخطوة 7: تحديد المجلد الصحيح

**هذه خطوة حاسمة!**

1. في الجانب الأيسر، ابحث عن مجلد اسمه:
   - **`public_html`** (الأكثر شيوعاً)
   - أو **`www`**
   - أو **`htdocs`**
   - أو اسم الدومين الخاص بك

2. اضغط **مرتين** على هذا المجلد لفتحه

3. **تنظيف المجلد (مهم!):**
   - إذا وجدت ملفات قديمة داخله (مثل index.html قديم)
   - حددها كلها (Ctrl + A أو Cmd + A)
   - اضغط زر **Delete** أو **"حذف"**
   - أكد الحذف

الآن المجلد فارغ ونظيف!

### الخطوة 8: رفع ملفات الموقع

**الآن الجزء المهم:**

1. **ارجع لجهازك** - افتح مجلد `build` الذي حضرناه في الخطوة 3
2. **افتح محتوياته** (ادخل داخل المجلد)
3. **حدد كل شيء داخل build:**
   - على Windows: Ctrl + A
   - على Mac: Cmd + A
   
   ستشاهد محدد:
   - index.html
   - مجلد static
   - asset-manifest.json
   - وملفات أخرى

4. **ارجع لـ File Manager في المتصفح**

5. **ابحث عن زر Upload** في الأعلى
   - عادة يكون أيقونة سهم للأعلى ↑
   - أو كلمة **"Upload"** أو **"رفع"**

6. **اسحب الملفات المحددة** من جهازك وأفلتها في نافذة File Manager
   - **أو** اضغط على زر Upload واختر الملفات

7. **انتظر حتى ينتهي الرفع**
   - ستظهر شريط تقدم (Progress Bar)
   - قد يستغرق 2-5 دقائق
   - **لا تغلق المتصفح أثناء الرفع!**

8. **تأكد من اكتمال الرفع:**
   بعد الانتهاء، يجب أن ترى في `public_html`:
   ```
   ✓ index.html
   ✓ مجلد static/
   ✓ asset-manifest.json
   ✓ manifest.json
   ✓ robots.txt
   ```

### الخطوة 10: ربط الدومين بالاستضافة (إذا لزم)

**إذا كان الدومين من Hostinger أيضاً:**

1. في قسم Domains، اضغط على الدومين
2. ابحث عن خيار **"Point Domain"** أو **"ربط النطاق"**
3. اختر الاستضافة الخاصة بك
4. احفظ التغييرات
5. انتظر 5-10 دقائق

**إذا كان الدومين من مكان آخر:**

سنحتاج تغيير Nameservers:
1. في لوحة Hostinger، اذهب لـ Domains
2. اضغط على دومينك
3. ابحث عن **"Nameservers"**
4. انسخ الـ nameservers (عادة شكلها):
   ```
   ns1.dns-parking.com
   ns2.dns-parking.com
   ```
5. اذهب لموقع مزود الدومين الأصلي
6. غيّر الـ nameservers هناك
7. احفظ
8. انتظر 24-48 ساعة

## المرحلة الخامسة: تفعيل SSL (HTTPS) (5 دقائق)

### الخطوة 11: تفعيل شهادة SSL المجانية

**مهم جداً لأمان الموقع ومصداقيته!**

1. في **hPanel**، ابحث عن قسم **"Security"** أو **"الأمان"**
2. اضغط على **"SSL"** أو **"شهادات SSL"**
3. ابحث عن دومينك في القائمة
4. اضغط على **"Install SSL"** أو **"تثبيت SSL"**
5. اختر **"Free SSL"** أو **"Let's Encrypt"** (مجاني)
6. اضغط **"Install"** أو **"تثبيت"**
7. انتظر 2-5 دقائق

**النتيجة:** سيصبح موقعك:
- ❌ http://yourdomain.com (غير آمن)
- ✅ https://yourdomain.com (آمن - قفل أخضر)

### الخطوة 12: تفعيل Force HTTPS (إجباري)

لضمان أن الزوار دائماً يستخدمون HTTPS:

1. في نفس صفحة SSL
2. ابحث عن خيار **"Force HTTPS"** أو **"إجبار HTTPS"**
3. فعّله (اجعله ON)
4. احفظ

## حل المشاكل الشائعة

### المشكلة 1: الموقع لا يفتح - Error 404

**السبب:** الملفات في مكان خاطئ

**الحل:**
1. ارجع لـ File Manager
2. تأكد أن ملف `index.html` موجود **مباشرة** داخل `public_html`
3. **ليس** داخل مجلد فرعي!

**الترتيب الصحيح:**
```
public_html/
├── index.html          ← هنا!
├── static/
│   ├── css/
│   └── js/
```

**الترتيب الخاطئ:**
```
public_html/
└── build/              ← خطأ!
    ├── index.html
    └── static/
```

### المشكلة 2: الموقع يفتح لكن بدون تنسيق (CSS)

**السبب:** مجلد static مفقود أو في مكان خاطئ

**الحل:**
1. تأكد من وجود مجلد `static` بجانب index.html
2. تأكد من رفع **محتويات** build وليس المجلد نفسه

### المشكلة 3: اللوجو لا يظهر

**السبب:** ملف اللوجو مفقود أو الرابط خاطئ

**الحل:**
1. افتح File Manager
2. ادخل على: `public_html/static/media/`
3. تأكد من وجود ملف اللوجو
4. إذا لم يكن موجود، ارفعه يدوياً

### المشكلة 4: الموقع يعمل لكن بدون HTTPS

**الحل:**
1. ارجع للخطوة 11
2. انتظر 10-15 دقيقة بعد تثبيت SSL
3. امسح cache المتصفح (Ctrl+Shift+Delete)
4. جرب مرة أخرى

### المشكلة 5: "DNS_PROBE_FINISHED_NXDOMAIN"

**السبب:** الدومين غير مربوط بشكل صحيح

**الحل:**
1. تحقق من ربط الدومين (الخطوة 10)
2. انتظر حتى 48 ساعة لانتشار DNS
3. اختبر الـ DNS على: https://dnschecker.org

### المشكلة 6: الموقع يعمل بدون www لكن لا يعمل مع www

**الحل:**
في File Manager:
1. ارجع لـ `public_html`
2. ابحث عن ملف `.htaccess`
3. إذا لم يكن موجوداً، أنشئه (سأعطيك الكود)
4. أضف هذا الكود:

```apache
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.(.*)$ [NC]
RewriteRule ^(.*)$ https://%1/$1 [R=301,L]
```

## نصائح إضافية

### 1. احتفظ بنسخة احتياطية
- قبل أي تعديل مستقبلي
- في File Manager، اضغط على مجلد public_html
- اختر "Compress" أو "ضغط"
- حمّل الملف المضغوط على جهازك

### 2. تفعيل Cloudflare (اختياري لكن موصى به)
- اذهب إلى cloudflare.com
- أنشئ حساب مجاني
- أضف دومينك
- اتبع التعليمات لتغيير Nameservers
- **المزايا:**
  - حماية من الهجمات
  - تسريع الموقع
  - Bandwidth غير محدود

### 3. مراقبة الزوار
أضف Google Analytics:
1. اذهب إلى analytics.google.com
2. أنشئ حساب وأضف موقعك
3. انسخ كود التتبع
4. في File Manager، افتح index.html
5. الصق الكود قبل `</head>`
6. احفظ
